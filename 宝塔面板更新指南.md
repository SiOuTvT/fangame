# 宝塔面板更新指南

既然你已经在宝塔面板上运行了项目，路径是 `/www/wwwroot/fangame-next`，这是专门为你准备的更新指南。

---

## 📋 更新前准备

### 1. 备份数据库（非常重要！）

在宝塔面板中：
1. 点击左侧菜单 **数据库**
2. 找到 `fangame` 数据库
3. 点击右侧的 **备份** 按钮
4. 等待备份完成

或者使用命令行：
```bash
cd /www/wwwroot/fangame-next
PGPASSWORD='你的数据库密码' pg_dump -U fangame -h localhost fangame > backup_$(date +%Y%m%d_%H%M%S).sql
```

### 2. 备份当前代码

在宝塔面板中：
1. 进入 **文件** → `/www/wwwroot/`
2. 右键 `fangame-next` 文件夹
3. 选择 **压缩**
4. 命名为 `fangame-next_backup_日期.zip`

---

## 🚀 更新步骤（推荐方法）

### 方法一：通过宝塔面板文件管理器（最简单）

#### 第1步：上传新文件

1. 在本地电脑上，打包以下文件和文件夹（**不要包含 node_modules**）：
   - `src/`
   - `prisma/`
   - `public/`
   - `package.json`
   - `package-lock.json`
   - `next.config.ts`
   - `tsconfig.json`
   - `components.json`
   - `.env` (确保包含正确的配置)
   - 其他配置文件

2. 压缩成 `fangame-next-update.zip`

3. 登录宝塔面板

4. 进入 **文件** → `/www/wwwroot/fangame-next/`

5. 点击 **上传** 按钮，上传 zip 文件

6. 上传完成后，右键 zip 文件 → **解压**

7. 选择 **覆盖现有文件**

8. 点击 **确定**

#### 第2步：安装依赖和重新构建

1. 在宝塔面板左侧菜单，点击 **终端** 或 **SSH**

2. 执行以下命令：

```bash
# 进入项目目录
cd /www/wwwroot/fangame-next

# 安装依赖
npm install

# 生成 Prisma 客户端
npx prisma generate

# 运行数据库迁移（会应用新的索引）
npx prisma migrate deploy

# 停止当前运行的服务
pm2 stop fangame-next

# 重新构建
npm run build

# 启动服务
pm2 start fangame-next

# 查看状态
pm2 status
```

#### 第3步：重启网站

在宝塔面板中：
1. 点击左侧 **网站**
2. 找到你的网站
3. 点击右侧的 **设置**
4. 点击 **重启**

---

## ⚠️ 常见问题

### 问题1：npm install 失败

**错误信息**：内存不足或权限错误

**解决方法**：
```bash
# 检查磁盘空间
df -h

# 清理 npm 缓存
npm cache clean --force

# 重新安装
npm install

# 如果还是失败，尝试增加交换空间
fallocate -l 2G /www/swapfile
chmod 600 /www/swapfile
mkswap /www/swapfile
swapon /www/swapfile
```

### 问题2：构建失败

**错误信息**：TypeScript 错误或内存不足

**解决方法**：
```bash
# 检查 Node 版本
node --version
# 应该是 v20.x

# 如果版本不对，在宝塔面板：
# 软件商店 → 搜索 "Node" → 安装 Node 20

# 清除 .next 缓存
rm -rf .next

# 重新构建
npm run build
```

### 问题3：数据库迁移失败

**错误信息**：连接失败或权限错误

**解决方法**：
```bash
# 检查 PostgreSQL 是否运行
systemctl status postgresql

# 测试数据库连接
PGPASSWORD='你的密码' psql -U fangame -d fangame -h localhost -c "SELECT 1;"

# 如果连接成功，重新运行迁移
npx prisma migrate deploy
```

### 问题4：网站无法访问（502 错误）

**原因**：PM2 进程未启动或端口被占用

**解决方法**：
```bash
# 检查 PM2 状态
pm2 status

# 如果进程不存在，重新启动
pm2 start npm --name "fangame-next" -- start

# 检查端口占用
lsof -i :3000

# 如果端口被占用，杀掉进程
kill -9 <PID>

# 重新启动
pm2 restart fangame-next
```

### 问题5：图片无法显示

**原因**：next.config.ts 中的域名白名单限制

**解决方法**：
检查 `.env` 文件中的 `NEXTAUTH_URL` 是否正确，并确保图片源在白名单中。

## 💡 小贴士

### 1. 定期备份

建议在宝塔面板设置定时任务：
1. 点击 **计划任务**
2. 添加新任务：
   - 任务类型：Shell 脚本
   - 执行周期：每天凌晨 2 点
   - 脚本内容：
     ```bash
     #!/bin/bash
     BACKUP_DIR="/www/backup/fangame"
     mkdir -p $BACKUP_DIR
     PGPASSWORD='你的密码' pg_dump -U fangame -h localhost fangame > $BACKUP_DIR/db_$(date +%Y%m%d).sql
     
     # 删除7天前的备份
     find $BACKUP_DIR -name "db_*.sql" -mtime +7 -delete
     ```

### 2. 监控日志

在宝塔面板：
1. 点击 **文件**
2. 进入 `/www/server/panel/logs/`
3. 查看 PM2 日志

或者实时查看：
```bash
pm2 logs fangame-next
```

### 3. 性能优化

如果服务器配置较低：
```bash
# 增加 Node.js 内存限制
pm2 delete fangame-next
pm2 start npm --name "fangame-next" -- start --max-old-space-size=2048
```
