## 一、背景与问题

### 1.1 当前问题
- 侧边栏"标签浏览"链接到 `/search?showTags=1`，但搜索页面无专门标签展示
- 用户无法浏览全站所有标签
- 无法按分类查看标签
- 无法了解每个标签下有多少游戏

### 1.2 页面定位
**独立的标签浏览页面** (`/tags`) 是一个：
- 以标签为核心的浏览入口
- 帮助用户通过标签发现游戏
- 展示标签体系和统计信息

**与搜索页面的区别**：
- 搜索页面：以"找游戏"为目的，标签是筛选工具
- 标签页面：以"逛标签"为目的，标签是浏览主体

## 三、核心功能模块

### 3.1 搜索标签
- 输入关键词实时过滤标签
- 支持标签名、别名搜索
- 防抖处理（300ms）

### 3.2 热门标签云
- 按使用频次排序 Top 20-30
- 标签大小/颜色深浅反映热度
- 点击跳转到 `/search?tag=标签名`
- 悬停显示 Tooltip：标签说明 + 游戏数量

### 3.3 分类浏览卡片
- 按标签组分类展示（作品类型、游戏类型、世界观、主题等）
- 每个分类展示该组下的热门标签（Top 5-8）
- 点击分类标题展开/收起

### 3.4 全部标签索引
- 按标签组分组展示全部标签
- 每个标签显示：名称 + 游戏数量
- 支持 A-Z 字母索引跳转（中文标签按拼音首字母）
- 支持 0-9 数字索引

### 3.5 标签统计面板（可选）
- 全部标签总数
- 已发布游戏总数
- 热门标签排行榜（带游戏数量）

## 四、数据结构

```typescript

interface TagBrowserPageData {
  // 热门标签（按使用频次）
  hotTags: Array<{
    id: string
    name: string

    gameCount: number

  // 标签组及其标签
  tagGroups: Array<{
    id: string
    name: string
    color: string
    description?: string
    tags: Array<{
      id: string

      color: string
      gameCount: number
    }>

  // 统计信息

    totalTags: number
    totalGames: number

  tagsByLetter: Record<string, Array<{
    id: string

    color: string
    gameCount: number
    group: { id: string; name: string }
  }>>
}
```

## 五、交互设计

| 交互 | 行为 |
|------|------|
| 点击标签 | 跳转到 `/search?tag=标签名`，带筛选结果 |
| 悬停标签 | 显示 Tooltip：标签说明 + 游戏数量 |
| 搜索标签 | 实时过滤下方标签列表（防抖 300ms） |
| 点击分类 | 展开/收起该分类下的标签 |
| 点击字母 | 滚动跳转到对应首字母的标签区域 |
| 滚动监听 | 滚动时高亮当前字母索引 |

## 六、技术实现要点

### 6.1 数据获取
- **SSR 渲染**：页面首次加载时获取全部标签数据，有利于 SEO
- **缓存策略**：标签数据不常变化，缓存 5-10 分钟
- **增量更新**：后台标签变化时可选择性刷新

### 6.2 游戏数量计算
- 统计每个标签关联的已发布游戏数（`isPublished: true`）
- 可通过 Prisma `count` 或预计算字段实现

### 6.3 中文拼音排序
- 使用 `pinyin-match` 或类似库处理中文标签首字母索引
-  fallback 到 Unicode 首字符

### 6.4 响应式设计
- 移动端标签云变为滚动横向列表
- 分类卡片变为堆叠布局
- 字母索引变为顶部固定 Tab 或底部悬浮条

### 6.5 性能优化
- 标签云使用 CSS `transform` 动画
- 大量标签时使用虚拟滚动
- 图片懒加载（如有标签图标）

## 七、与搜索页面的协作

```
标签页面 (/tags)           搜索页面 (/search)
      │                         ▲
      │ 点击标签                │ 可查看标签筛选
      ▼                         │
/search?tag=xxx ────────────────┘
```

- 标签页是"逛"的入口
- 搜索页是"找"的工具
- 点击标签页的标签 → 带筛选参数跳转到搜索页
