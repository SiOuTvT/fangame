## 模式驱动全项目整改（Pattern-Driven Remediation，模式A–L 全部落地）
用户核心指令：每个问题当作"模式"处理——全项目搜索同类实现，一次性统一修复并再次扫描确认零遗漏；不保留明显有缺陷的旧实现；重构落后/重复/历史遗留为统一最佳实践。

### 已建立的 Single Source of Truth 收敛点
- `src/lib/api-handler.ts`：统一 `withHandler`（捕获 AppError/ZodError/PrismaKnownError→标准响应）、`safeParseJson`（非法 JSON→422）。移除死代码 `parseBody`/`parseSearchParams` 及未用 `z` 导入；`RateLimitError.retryAfter` 现已写入 `Retry-After` 头（L2③）。
- `src/lib/date.ts`：`toShanghaiDate`/`shiftShanghaiDate` 统一时区（M5/M18/M-C）。
- `src/lib/permissions.ts`：`isSuperAdminRoute`/`hasRole`/`ROLE_LEVEL` 统一权限（H1/M7/M8、middleware 去硬编码路由表）。
- `src/lib/password.ts`：`validatePassword` 统一密码强度（M14，覆盖 register/resetPassword/setup）。
- `src/lib/sanitize.ts`：`sanitizeUrl` 统一用户可控 URL 校验（M6/M19）。
- `src/lib/emoji.ts`：抽取共享 `EMOJI_LIST`（L1，消除两处重复数组）。
- `src/lib/storage.ts`：`deleteByUrl(url)` 按 URL 反推 key 删除，清理孤儿文件（L10）。

### 关键修复
- 模式A：59 个 API 路由 `req.json()`→`safeParseJson`（统一 422 契约）；Prisma 错误映射 P2002→Conflict/P2025→NotFound 等。
- 模式B：admin 用户 PUT 改为 `requireAdminRole("SUPER_ADMIN")`；middleware 用 `isSuperAdminRoute` + 真实 `x-forwarded-proto` 派生 HSTS。
- 模式D/F：sanitizeUrl 覆盖 avatar/banner/forum imageUrl/creator/announcement/avatar-frame；后台删收藏回退 `favoriteCount`（事务）。
- 模式E：RedisCache.clear() 真实 SCAN+pipeline 删除 `fangame:*`。
- 模式G：NextAuth v5 `authorize(credentials, request)` 接 `checkRateLimit`（按 IP）。
- 模式H：管理员删帖/删评路由补传 `isAdmin`。
- 模式J：next.config 移除伪造 `X-Forwarded-Proto` 头，加 `server.trustProxy: true`。
- 模式K：setup 路由改用 `validatePassword`、bcrypt cost 统一 12。
- 模式L（技术债）：L2 死代码移除+文档同步；L6 admin 审计/文件清理 `.catch(()=>{})`/`catch{}` 全部改为 `logger` 记录；L9 防删除/降级最后一名 SUPER_ADMIN（repo 加 `countSuperAdmins`）；L7 `reactStrictMode: true`；L3 登出 `localStorage.clear()`→只删 `local_avatar`/`checkin_status`（保留主题/NSFW），game-detail intro tab 补 `role=tabpanel`；L4 浏览量回退改默认 0；L8 复核确认无客户端 UploadThing 直传，`connect-src` 维持最小权限（不补 utfs.io）。
