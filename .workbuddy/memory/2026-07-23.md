# 2026-07-23 工作日志

## 全站 UI 资源完整性巡检（Production Asset Audit）

背景：生产环境后台「空状态」页面 Lucide 图标渲染成原始字符串（Map / Lock / Image / BookOpen / MessageCircle / Inbox / CheckCircle / Package …）。

### 根因（已实证）
`src/lib/emoji-icons.ts` 旧实现用 `const MAP = { Heart, CheckCircle, ... }` 对象字面量 + `MAP[name]` 动态查表。
生产构建下（Turbopack/webpack usedExports + lucide-react `sideEffects:false`），这些"仅被动态 key 读取"的属性被 Tree-Shaking/DCE 整组删除，
导致 `getIconForEmoji` 返回 null → `EmotionalIcon` 回退到 `<span>{name}</span>`（渲染字符串）。
实证：旧产物 `.next/static/chunks` 中 grep `BookmarkPlus`（仅被 emoji-icons.ts 使用的图标）未命中 → 证明注册属性被 DCE 删除。Dev 不压缩故正常。

### 修复（同类型一起修）
1. `src/lib/emoji-icons.ts`：改 `switch(name){ case "X": return X }` 静态返回，每个组件在可达路径被静态引用，无法被 DCE；导出静态 `EMOJI_OPTIONS`（`as const`）供 admin 下拉；新增 `Cloud`。
2. `src/components/tag-position-checkbox.tsx`：删除 `ICON_MAP` + lucide 直接 import，改用 `getIconForEmoji(pos.icon)`（同一动态按名查表模式的同类隐患）。
3. `src/app/layout.tsx`：删除无用的 `notoSans = { variable: "" }` 占位；同时修掉删除后残留的两处 `${notoSans.variable}` 引用（避免引入 TS 编译错误）。
4. `public/icon-192.png` + `public/icon-512.png`：新增 PWA 图标（用 `next/og` 的 ImageResponse 生成，与 `icon.tsx` 同款 🎮 设计）；`manifest.ts` 原已引用这两个路径，404 修复。生成脚本留 `scripts/gen-pwa-icons.cjs`。

### 维度结论
- 一、Lucide 图标：1 个真实缺陷（DCE）已修，同类隐患 tag-position 已修。
- 二、Emoji：0 缺陷（全部 unicode 文本渲染，无外部 CDN emoji 图，统一经 EmotionalIcon）。
- 三、字体：0 缺陷（系统字体栈 PingFang/YaHei/Source Han，无 next/font/google 或 Google CDN link；删除无用 notoSans）。
- 四、SVG 与 Icon：0 新增缺陷（PWA 图标 404 属静态资源维度，已修）。
- 五、静态资源：1 个真实缺陷（manifest 引用缺失的 icon-192/512.png）已修。
- 六、部署差异：Dockerfile 正确 copy public/ 进 standalone 镜像；DCE 是唯一 dev≠prod 差异，已修。
